# Week 3 - Server Architecture & Databases 🗄️

This week, we stepped into backend development! We learned how to build a web server using Node.js and Express, and how to connect it to a MongoDB database.

---

## 📚 What We Learned
- **Database Basics:** Understanding NoSQL databases, collections, and documents.
- **CRUD Operations:** Practicing Create, Read, Update, and Delete operations in MongoDB using the mongo shell.
- **Express Servers:** Setting up an Express.js server to handle incoming HTTP GET and POST requests.
- **Cloud Database:** Connecting our application to a remote MongoDB Atlas database cluster.

---

## 📂 Directory Index
- **`MongoDbPractice.txt`**: A log of raw MongoDB queries and commands run in the terminal.
- **`server.js`**: The main entry point for our Express server.
- **`models/`**: Folder containing our Mongoose schemas (e.g., user and product models).
- **`reqq.http`**: A file to test our server endpoints using a REST client tool.
